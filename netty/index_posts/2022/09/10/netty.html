NIO & BIO

Andy Sun 10 Sep 2022

1, stream vs channel

  • stream means socket stream or file input oupt stream
  • channel means netty socket channel
  • stream does not buffer data , channel will use sending buffer area , receiving buffer area , channel call system api
  • stream only support blocking api , channel support both blocking ,unblocking
  • Stream and channel are both full duplex, they can read and write at same time

3,sync & async

Sync : current thread get result itself . Blocking IO , UnBlocking IO ，Multiplexing(single thread+select())

4，zero copy

A, tranditional io

Read data from disk

File f = new File("helloword/data.txt");
RandomAccessFile file = new RandomAccessFile(file, "r");

byte[] buf = new byte[(int)f.length()];
file.read(buf);

Socket socket = ...;
socket.getOutputStream().write(buf);

the process is blow:

  1. java are not able to read or write IO，so after reading，the process will change from java user mode to kernel mode to call api of kernel，write the data into buffer。user thread is blocked during this period .

  2. Change from kernel mode to user mode , ready data(byte[] buf) from kernel mode into user mode

  3. Call write ，write data into socket buffer from user buffer .

  4. java cannot write data into network card directly, so we need to change from user mode to kernel mode

  • mode changes happen 3 times
  • Data copy happen 4 times
