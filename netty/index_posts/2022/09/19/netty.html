1）CONNECT_TIMEOUT_MILLIS

  • This is for SocketChannal (client)

  • when the client create connection , it will throw timeout exception if it is not connected to server in timeout time

  • SO_TIMEOUT is mainly used in blocking io
@Slf4j
public class TestConnectionTimeout {
    public static void main(String[] args) {
        NioEventLoopGroup group = new NioEventLoopGroup();
        try {
            Bootstrap bootstrap = new Bootstrap()
                    .group(group)
                    .option(ChannelOption.CONNECT_TIMEOUT_MILLIS, 300)
                    .channel(NioSocketChannel.class)
                    .handler(new LoggingHandler());
            ChannelFuture future = bootstrap.connect("127.0.0.1", 8080);
            future.sync().channel().closeFuture().sync();
        } catch (Exception e) {
            e.printStackTrace();
            log.debug("timeout");
        } finally {
            group.shutdownGracefully();
        }
    }
}

2）SO_BACKLOG

  • belong the setting of ServerSocketChannal

    sequenceDiagram participant c as client participant s as server participant sq as syns queue participant aq as accept queue s ->> s : bind() s ->> s : listen() c ->> c : connect() c ->> s : 1. SYN Note left of c : SYN_SEND s ->> sq : put Note right of s : SYN_RCVD s ->> c : 2. SYN + ACK Note left of c : ESTABLISHED c ->> s : 3. ACK sq ->> aq : put Note right of s : ESTABLISHED aq -->> s : s ->> s : accept()

  1. First handshake，client send SYN to server，its status is changed to SYN_SEND ;

    server received，its status is changed to SYN_REVD，and put this request into sync queue

  2. Second handshake，server reply SYN + ACK to client ;

    client got it，its status is changed to ESTABLISHED，and send ACK to server

  3. Third handshake，server got ACK，its status is changed to ESTABLISHED，put this request from sync queue to accept queue

Attention

  • Before linux 2.2 ，backlog contain two queue , after 2.2 ，set them sepratly as below:

  • sync queue - half connection queue
    • It is set by /proc/sys/net/ipv4/tcp_max_syn_backlog ，if syncookies is inuse ，this setting would be ignored
  • accept queue - complete connection queue
    • It is set by /proc/sys/net/core/somaxconn ，core will get the less value if use listen method
    • accpet queue is full，server send refusing connecting messasge to client

netty set the value by option(ChannelOption.SO_BACKLOG, value)

the source code is

public class DefaultServerSocketChannelConfig extends DefaultChannelConfig
